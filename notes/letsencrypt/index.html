      Let’s Encrypt #

      Obtain wildcard certificate #

      Use the manual plugin and the DNS challenge to obtain a wildcard certificate.

      certbot certonly --manual --preferred-challenges dns -d '*.example.com'

      Create the TXT record as instructed by certbot. Before continuing, use dig to confirm the records is applied.

      dig txt _acme-challenge.example.com

      When dig shows that the record is applied, continue the certbot process.

      Google Dig If you don’t have dig installed, you could use the Dig tool in the Google Admin Toolbox.

      Renew manual plugin certificates #

      To “renew” certificates generated with the manual plugin, run the same command that you ran to generate the certificate initially.

      New private key This approach creates a new private key as well as a new certificate. You will need to replace both.

      For example, to “renew” the certificate generated in Obtain wildcard certificate, run the same command again.

      certbot certonly --manual --preferred-challenges dns -d '*.example.com'
